# Introduction

This document demonstrates mdtexpdf's ability to handle multiple languages and scripts. When the document contains CJK (Chinese, Japanese, Korean) characters, mdtexpdf automatically switches to XeLaTeX with appropriate fonts.

---

# Conclusion

This document demonstrates that mdtexpdf can handle:

1. **Latin scripts** with diacritical marks (French, German, Spanish, etc.)
2. **Cyrillic scripts** (Russian, Ukrainian)
3. **CJK characters** (Chinese, Japanese, Korean)
4. **Devanagari** (Hindi)
5. **Arabic script** (Arabic, Hebrew)
6. **Greek** (with mathematical symbols)

When CJK characters are detected, mdtexpdf automatically:
- Switches to XeLaTeX engine
- Loads appropriate CJK fonts
- Maintains proper character spacing

---

## Technical Notes

For best results with CJK content:

1. Ensure you have CJK fonts installed:
   ```bash
   # Ubuntu/Debian
   sudo apt-get install fonts-noto-cjk
   ```

2. The xeCJK package must be available:
   ```bash
   sudo apt-get install texlive-lang-chinese
   ```

3. For Arabic/Hebrew (RTL languages), additional configuration may be needed.
